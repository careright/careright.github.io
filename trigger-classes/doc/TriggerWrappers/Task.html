Overview

Represents a task record.

The task type controls which fields are required: <ul> <li>If User Assignment is “required”, then for_user attribute is required</li> <li>If Location Assignment is “required”, then location attribute is required</li> <li>If Patient/Contact Assignment is “required” then person attribute is required</li> </ul>

If any of the assignment options are set to “None”, then assignment to the wrapper class will be ignored.

If start_date (and optionally due_date) is set, then a followup will be created in the system linked to the new task.

Examples:

Building a task object

// Tasks can be created via the builder helpers:
var task = builder.task.build('CODE')

// Or via an existing patient object:
var task = patient.create_task('CODE')

Task Creation

var task = this.patient.create_task('BILLING')
task.note = 'remember to do some billing stuff'
task.location = this.patient.location

// create a followup to start tomorrow, due next week
task.start_date = builder.date.today().add_days(1)
task.due_date = task.start_date.add_days(6)

if (!task.save()) {
  script.fatal('save failed')
}
