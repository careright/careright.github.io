#each(options, callback) ⇒ Object

Searches for invoices.

The function callback is called for each invoice which matches the criteria given in options.

Valid options: <ul> <li>account_held [String] filters invoices according to the

<code>Hold Statements</code> flag of their account; one of:
<ul>
<li><code>all</code> (default) - All invoices regardless of hold
statements</li>
<li><code>yes</code> - Only invoices of accounts holding
statements</li>
<li><code>no</code> - Only invoices of accounts not holding
statements</li>
</ul>

<li>account_type [String] one of:

<ul>
<li><code>private</code> - Only invoices on private accounts will be
found</li>
<li><code>loan</code> - Only invoices on loan accounts will be
found</li>
</ul>

</li> <li>extra_code_1 [String] Invoice custom extra code 1 has the supplied value (similarly for code 2..4)</li> <li>extra_code_1_not [String] Invoice custom extra code 1 does not have the supplied value (similarly for code 2..4)</li> <li>has_no_opening_balance [Boolean] if true, only include invoices which do not have an opening balance statement on them</li> <li>has_no_receipt [Boolean] if true, only include invoices which do not have a receipt allocated to them</li> <li>processed [Boolean] if true, only include invoices which have been processed and accepted by Medicare</li> </ul>

Examples:

Iterate over all loan accounts which have been processed

finder.invoice.each({ account_type: 'loan', processed: true }, function (invoice) {
  console.log('Found invoice ' + invoice.id)
})
