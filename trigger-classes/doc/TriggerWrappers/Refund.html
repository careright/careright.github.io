#auto_allocate(options = {}) ⇒ Boolean

Automatically allocate this Refund.

The credit amount of this Refund will be allocated to line items which are not fully paid. Line items will be paid down in reverse chronological order.

If the Refund balance is higher than the amount on outstanding line items, then the last line item will be overallocated (if the overallocate options is true) or allocation will fail (default).

Valid options: <ul> <li>overallocate [Boolean] if true allow overallocation</li> </ul>

Parameters:

  • options (Object) (defaults to: {})

Returns:

  • (Boolean) —

    true if no issues were encountered

#reallocate(allocations) ⇒ Boolean

Allocates (or reallocates) payments from this Refund to a set of line items.

allocations must be an array where each element is a pair of two items: <ul> <li>a LineItem object, and<li> <li>an amount to allocate (a decimal number or a string containing a number)</li> </ul>

The affected objects will be saved automatically.

Examples:

Allocate an amount equal to paid rebate to each line item for an invoice

var line_items_and_amounts = []
invoice.line_items.forEach(function (invoice) {
  line_items_and_amounts.push([line_item, line_item.rebate_paid_by_medicare])
})
if (refund.reallocate(line_items_and_amounts)) {
  console.log('Reallocation successful')
}

Parameters:

  • allocations (Array) —

    described above

Returns:

  • (Boolean) —

    true if the reallocation succeeded
