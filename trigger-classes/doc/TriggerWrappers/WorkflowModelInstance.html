Overview

Represents a specific instance of a workflow model.

Each attribute added to the associated trigger class (of type Workflow Model) is available as a property of this class.

In trigger events related to Workflows, the current workflow is made available to trigger scripts as a predefined variable named workflow.

When an assessment is approved the elements in the assessment with the same name and data type are written to the Workflow Model this will overwrite and previously set values. If you use a trigger script in a trigger button in an assessment then only modify Workflow Model attributes NOT included on the assessment or the changes will be over written. The Trigger Script executed on approval will already have the changes written from the assessment to the Workflow model so if you want to change a value you can and it will not be overwritten.

Examples:

workflow.favorite_cake = 'Chocolate'
if (!workflow.save()) {
  console.log('Failed to save workflow')
}
// Set The workflow transition action based on the value of an assessment
// element. (This will only work when approving an Assessment used in a
// Workflow)
workflow.next_action = assessment.decision
