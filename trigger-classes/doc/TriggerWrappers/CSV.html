Overview

This class is used to construct CSV (comma separated values) data.

There are 2 ways to prepare CSV data: <ul> <li>By defining fields and (optionally) their defaults. This is useful when constructing CSV files with a large number of fields and if some have a constant value.</li> <li>By adding each row individually as an array. This is useful for very small CSV files</li> </ul>

To retrieve the entire CSV contents, use the to_s method. If running in an export context, this object can be passed directly to export_file which is more efficient.

This class will correctly quote fields which contain spaces, commas and quotation characters.

Examples:

Defining fields and defaults

var csv = builder.csv.new()
csv.set_fields(['First', 'Second', 'Third', 'Fourth'])
csv.set_defaults({ First: 1, Second: 2 })
csv.add_header()
csv.add_data({ First: 'AAA', Third: 'CCC' })

// Resulting CSV data will be:
// First,Second,Third,Fourth
// AAA,2,CCC,

Direct building without field definitions

var csv = builder.csv.new()
csv.add_row(['First', 'Second', 'Third', 'Fourth'])
csv.add_row(['AAA', 2, 'CCC', null])

// Result is the same as the previous example

Instance Method Details

#add_data(data) ⇒ Object

Appends a row to the contents.

Key names must match names previously provided to set_fields.

If a defined field is missing from data, then a default previously provided to set_defaults will be used.

If a defined field is missing from data and has not had a default provided, it will be stored as an empty value.

Parameters:

  • data (Object)

#add_header(_dummy) ⇒ Object

Appends a header row to the contents.

This method is typically used once before adding any data if the CSV requires a header row.

#add_row(data) ⇒ Object

Appends a row of values to the contents.

A row will be appended to the contents with as many elements as the provided array, without reference to field definitions.
